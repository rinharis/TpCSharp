1 Eclipse

Installation

Eclipse est un outil de développement Java gratuit, il fonctionne sous Windows, Unix.

Il a besoin d'un JDK pour fonctionner, c'est un outil fait en Java. La version du Java nécessaire à Eclipse n'est pas obligatoirement la même que celle qui vous servira pour vos développements, lisez bien les près-requis de la version Eclipse choisie.

Récupérez la version adaptée à votre système (Windows, Unix, Mac ...)

Téléchargez la version adaptée à votre type de système 32 ou 64bits.

Prenez la version la plus adaptée à vos développements, dans le doute prenez la version Eclipse IDE for Java EE Developers qui permet de faire des applications Webs.

Téléchargement disponible ici http://www.eclipse.org/downloads/

Il n'y a pas d'installation (fichier exe, msi, ...) à proprement parler, il suffit de dé zipper l'archive ZIP sur votre disque dur. A besoin, installez 7-zip

Important : sous windows, il est conseillé de placer Eclipse directement au niveau de la racine d'une partition, par exemple sur c: (limitation sur la taille des noms de fichiers).

Vous devriez avoir l'arborescence suivante :

Paramétrage

Lancez Eclipse, double cliquez sur eclipse.exe

Eclipse va vous demander où créer votre workspace (espace de travails), c'est là qu'il mettra votre code.

Allez dans le menu Window/Preferences, puis tapez encoding dans le filtre en haut à gauche.

Si vous n'êtes pas en UTF-8, placez-vous en UTF-8 pour Workspace, CSS, HTML et JSP.

Faites Apply (en bas à droite), puis effacez votre filtre (encoding), allez sur Java/Installed JREs

Vérifiez que vous pointez bien sur un JDK et pas un JRE. Le cas échéant modifiez votre chemin afin de pointer vers un JDK.

N'oubliez pas de faire Apply.

2 Projet Standard

Création d'un projet

Avant de créer du code Java, il faut créez un nouveau projet Java dans lequel on pourra placer notre code

  • File/New/Java Project

Donnez-lui un nom, par exemple Exo01, vous pouvez regarder les différents écrans du Wizard de configuration, mais nous ne modifierons aucun paramètre.

Vous passerez en perspective Java (et non Java EE).

Création d'une classe

Créez une nouvelle classe Java

  • File/New/Class

Donnez-lui un nom de package, par exemple : fr.exo01

Donnez-lui un nom de classe, par exemple SecondExo

Reprenez le code de la méthode main de l'exo00 et placez le dans votre classe.

Compilation

Sauvegardez votre code (ctrl + s ou via le menu faites usage des icônes qui ressemblent à des disquettes)

Il n'y a pas de compilation à proprement parler dans Eclipse, pour compiler il vous suffit de sauvegarder vos fichiers.

Lancement

Comme nous avons réalisé une classe qui possède une méthode main, nous pouvons l'exécuter.

Faites un clic droit sur votre fichier SecondExo

Popup menu Run As/Java Application

Livrable

Si vous voulez faire un livrable de votre projet, faites un clic droit dessus puis Export -> Jar File ou Runnable Jar File s'il y a une classe avec un main.

Suivez les écrans du Wizard, au final vous obtiendrez un JAR (Java Archive)

Pour exécuter un Jar avec un main : 

java -jar nomDuJar.jar

3 Projet Maven

Présentation Maven

L'outil Maven permet de paramétrer un projet de manière neutre (c.a.d qui n'est pas associé à un outil de développement spécifique).

Les projets de cette formation sont réalisés en Maven afin de vous familiariser avec cette technique qui est la plus répandue actuellement.

Toutes les librairies qui seront nécessaires à nos projets seront automatiquement téléchargées d'internet grâce à Maven.

Toutes les informations associées aux paramétrages de votre projet seront dans le fichier pom.xml qui se trouve à la racine du projet.

La balise XML <artifactId> dans votre fichier pom.xml doit correspondre au context root de votre application WEB. Le context root est le nom qui apparait dans l'URL juste après celui du serveur J2EE.

Si vous modifiez le fichier pom.xml, vous devez faire un clic droit sur votre projet puis Maven - Update Project... afin qu'Eclipse prenne en compte les modifications.

L'ensemble des librairies nécessaires seront placées sur votre disque dur dans le dossier [utilisateur]/.m2. Dans de rare cas, si Maven ne marche plus, pensez à l'effacer.

Eclipse intègre Maven, il n'y a rien à installer de particulier. Par contre si vous voulez utiliser Maven en ligne de commande il faudra l'installer.

Création du projet

Avant de créer du code Java, il faut créez un nouveau projet Java dans lequel on pourra placer notre code

  • File/New/Maven Project

Gardez les options standards dans le wizard, nous allons de toute manière les écraser plus tard.

Pour le groupId, indiquez un nom de package racine, par exemple ici : fr.exo01

Pour le artifactId, indiquez un nom de projet, par exemple ici : exo01.bonjour

groupId:artifactId représente en Maven un nom unique pour votre projet

Une fois le projet créé, copiez le fichier pom.xml qui se trouve à la racine de l'énoncé de l'exercice et collez le dans votre Eclipse à la place de celui généré.

Vous pouvez si vous le désirez, changer le groupId dans le pom.xml en indiquant un nom de package racine, par exemple ici : fr.exo01

Vous pouvez aussi changer le artifactId dans le pom.xml en indiquant un nom de projet, par exemple ici : exo01.bonjour. Attention sous Eclipse à garder la correspondance nom projet eclipse = artifactId

Important : Après toutes modifications faites sur le fichier pom.xml, faites un clic droit sur votre projet puis Maven - Update Project ...

Présentation du projet

L'arborescence d'un projet Maven est standardisée de la manière la suivante :

  • votre code web (qui touche à la présentation) (.jsp, .html, .css, .js, Images, ...) ira dans src\main\webapp
  • votre code Java (entre autre vos Servlet) (fichiers .java, ...) ira dans src\main\java
  • vos fichiers de configurations (log, properties, ...) iront dans src\main\resources
  • vos librairies de code (fichiers .jar) devront être déclarées en tant que dépendances Maven
  • votre fichier web.xml est dans src\main\webapp\WEB-INF
  • votre code de test Java (vos tests JUnit) (fichiers .java) ira dans src\test\java
  • vos fichiers de configurations pour le test (log, properties, ...) iront dans src\test\resources

Le dossier target sert de dossier temporaire aux actions de Maven.

Création d'une classe

Créez une nouvelle classe Java dans le dossier src/main/java

  • File/New/Class

Donnez-lui un nom de package, par exemple : fr.exo01

Donnez-lui un nom de classe, par exemple SecondExo

Reprenez le code de la méthode main de l'exo00 et placez le dans votre classe.

Compilation

Sauvegardez votre code (ctrl + s ou via le menu faites usage des icônes qui ressemblent à des disquettes)

Il n'y a pas de compilation à proprement parler dans Eclipse, pour compiler il vous suffit de sauvegarder vos fichiers.

Sous vous souhaitez compiler avec Maven, faites un clic droit sur votre fichier pom.xml puis Run As - Maven Build

Indiquez dans l'élément Goals clean compile

Après le succès du goal, Maven placera le résultat dans le dossier target.

De temps en temps, Eclipse et Maven se mélange, il vous faudra lancer un Project/Clean via Eclipse.

Lancement

Comme nous avons réalisé une classe qui possède une méthode main, nous pouvons l'exécuter.

Faites un clic droit sur votre fichier SecondExo

Popup menu Run As/Java Application

Livrable

Notre classe ayant une méthode main, si ce n'est pas déjà fait, indiquez dans le fichier pom.xml le nom long de la classe comme valeur de la balise <mainClass>. Un nom long de la classe = nom.de.package.NomDeLaClasse (ici : fr.exo01.SecondExo).

Comme indiqué plus haut, vous venez de modifier le fichier pom.xml, faites un clic droit sur votre projet puis Maven - Update Project ...

Sous vous souhaitez réaliser un livrable avec Maven, faites un clic droit sur votre fichier pom.xml puis Run As - Maven Build...

Indiquez le goal package dans la case Goals

Après le succès du goal, votre JAR sera dans le dossier target, pensez à faire un clic droit refresh afin de voir le résultat.
