1 Suite du projet

Interface ICompteRemunere

Toujours dans le ProjetBank, créez une nouvelle interface fr.banque.ICompteRemunere qui respectera les contraintes suivantes :

  • Méthodes (au moins) :
    • public double calculerInterets()
    • public void verserInterets()
    • public double getTaux()
    • public void setTaux(double unTaux)

Votre classe fr.banque.CompteRemunere doit implémenter cette interface (mot clef implements).

Interface ICompteASeuil

Toujours dans le ProjetBank, créez une nouvelle interface fr.banque.ICompteASeuil qui respectera les contraintes suivantes :

  • Méthodes (au moins) :
    • public void retirer(double uneValeur)
    • public double getSeuil()
    • public void setSeuil(double unSeuil)

Votre classe fr.banque.CompteASeuil doit implémenter cette interface (mot clef implements).

Classe CompteASeuilRemunere

Toujours dans le ProjetBank, créez une nouvelle classe fr.banque.CompteASeuilRemunere qui respectera les contraintes suivantes :

  • Cette classe hérite de la classe fr.banque.CompteRemunere ou fr.banque.CompteASeuil. Il vous appartient de décider.
  • Cette classe implémente fr.banque.ICompteRemunere et/ou fr.banque.ICompteASeuil. Il vous appartient de décider. Le choix sera dépendant du choix de votre classe parent
  • Attributs : seuil ou taux selon votre choix d'héritage
  • Constructeurs : au choix
  • Méthodes en fonction de l'héritage.

Par exemple, ici on a choisit d'hériter de CompteRemunere

Et Java 8 ?

En Java 8 vous pouvez ajouter du code dans les méthodes de vos interfaces (utilisation du mot clef default)

Nous pourrions ainsi coder dans les interfaces ICompteASeuil et ICompteRemunere les méthodes (en plus du prototypage).

Pour que cela fonctionne, nous devrions ajouter une interface ICompte qui sera l'interface mère de ICompteASeuil et ICompteRemunere.

Test

Dans votre classe fr.Run complétez votre méthode main, ajoutez des CompteASeuilRemunere.

Modifiez ou vérifiez que votre code qui permet de verser les intérêts sur tous les comptes de type rémunéré fonctionne aussi avec les CompteASeuilRemunere (faites usage de l'interface fr.banque.ICompteRemunere au lieu de la classe)
